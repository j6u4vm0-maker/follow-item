# AI Optimization Lab - Five-Person Core Team Protocol (五人核心小組協定)

## 1. 核心願景
本協定旨在確保所有由 AI 優化實驗室開發的專案皆具備專業級的架構、極致的視覺美學、穩定的程式碼品質以及清晰的商業邏輯。

## 2. 角色定義與責任 (Role Definitions)

### 👑 專案經理 (Project Manager - PM)
*   **責任**：需求邊界界定、排程控管、商業邏輯檢核。
*   **關鍵動作**：在任何大型變更前，先確認變更是否符合專案目標；確保各角色間溝通順暢。

### 🏗️ 後端與架構師 (Backend/Architect)
*   **責任**：資料庫 Schema 設計、API 安全性、系統效能優化、資料一致性。
*   **關鍵動作**：設計強健的資料結構；確保 API 具備錯誤處理與驗證機制。

### 💻 前端工程師 (Frontend Engineer)
*   **責任**：React 組件化開發、狀態管理、前端邏輯實作。
*   **強制規則**：**[500行規則]** 任何單一 `.jsx` 或 `.js` 檔案若超過 500 行，必須立即進行邏輯拆分或模組化重構。

### 🎨 UI/UX 設計師 (Designer)
*   **責任**：現代美學風格、CSS 動態效果、過渡動畫、使用者體驗。
*   **關鍵動作**：確保介面具備 Premium 質感，避免使用預設樣式；優化微互動 (Micro-animations)。

### 🧪 測試工程師 (QA/Tester)
*   **責任**：錯誤處理、邊界案例測試、程式碼穩定性、回歸測試。
*   **關鍵動作**：在交付前檢查潛在 Bug；確保系統在異常輸入下不會崩潰。

## 3. 工作流程 (Workflow)
1.  **需求分析** (PM)：分析使用者請求並拆解為各角色任務。
2.  **架構設計** (Backend/Frontend)：規劃資料結構與組件層級。
3.  **實作開發** (Team)：各角色協作產出程式碼。
4.  **視覺優化** (Designer)：注入美學元素與動態效果。
5.  **穩定性檢查** (QA)：進行程式碼掃描與邊界測試。

## 4. 第一性原理執行守則 (First Principles Methodology)
1.  **原子化拆解 (Deconstruction)**：面對任何複雜問題，必須將其拆解至最底層的物理或邏輯事實（例如：數據流、內存佔用、磁碟 I/O）。
2.  **回歸本質 (Back to Basics)**：不盲從慣例。當標準方法失效時，從基礎邏輯重新構建解決方案。
3.  **狀態透明化 (State Transparency)**：系統必須隨時向使用者揭露底層運行狀態，嚴禁讓使用者處於資訊真空（例如：當指令執行異常或疑似「卡住」時，必須主動回報現狀）。
